Cento Improvement Recommendations

Full codebase review · 2026-04-29 · 10 prioritised findings

TOP PRIORITY — Breadcrumb navigation across all agent-work HTML

Every HTML page in workspace/runs/agent-work/ is a dead end — no nav back to start-here.html or any parent. Agents generate these pages independently and none add breadcrumbs. Needs a shared standard: at minimum a one-line <nav class="breadcrumb"> in every generated HTML that links back to the run's start-here.html. Should be enforced via a template or generator rule so future agents can't skip it. Also affects runs with no start-here.html at all (e.g. run #16, industrial-panels-v1) — those runs need a hub page too.

1. Wire BriefGenerator to Claude

daily_tui.go — The BriefGenerator interface is a stub. Pipe gather_context.py output into a Claude API call. Scaffolding is complete; only the real implementation is missing.

2. Bridge failures are silent

bridge.sh — run_remote_node swallows SSH errors; the cluster TUI shows "online" on stale state. Add reachability timestamps to heartbeats so the TUI can show "online (last seen 12m ago)" vs "unreachable".

3. agent_work needs a no-Redmine fallback

agent_work.py — 1,815 lines requiring a live Redmine Postgres container over SSH. Add a flat-file JSON backend in workspace/runs/agent-work/. The status flow (Queued → Running → Review → Done) maps cleanly to local JSON.

4. Workspace cleanup command

workspace/runs/ has no rotation policy — logs grow forever, scan-onepager/ archives manually, cluster job dirs pile up. Add cento workspace clean [--older-than 30d] [--dry-run], registered in data/tools.json.

5. Industrial OS needs shared state

7 files (industrial_aux_tui.go, industrial_cluster_tui.go, industrial_jobs_tui.go, industrial_panel.py, etc.) each poll independently. A single ~/.local/state/cento/industrial-os/state.json keeps them in sync without extra IPC.

6. Live status bar in cento interactive

cento_interactive.go is static docs. Embed a compact cluster status bar at the bottom — one line, green/red dots per node, auto-refreshes. Reuse loadSnapshot from industrial_cluster_tui.go.

7. Python modules import each other unsafely

dashboard_server.py does sys.path.insert hacks to import from network_web_server and jobs_server. Extract shared logic (cluster_snapshot, job_summary, port utilities) into scripts/lib/cluster_data.py.

8. make check is syntax-only

Makefile runs go build, python -m py_compile, and bash -n — none catch runtime assumptions. Add make smoke: runs cento cluster status, cento network --web --no-open, and cento bridge mesh-status with a 5s timeout each.

9. Heartbeat should push, not poll

cluster_heartbeat() requires a manual query. A systemd timer (every 5 min) on Mac publishing to the relay would make heartbeat state live in the TUI. Extend cento bridge heal — which already installs services — to install the timer too.

10. Register agent_work in tools.json

agent_work.py is absent from data/tools.json. That means cento interactive, cento docs, and cento tools are blind to the most complex new addition. Free win: add the entry with subcommand examples.
