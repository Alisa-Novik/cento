iPhone Cento App

Manager report for Redmine #16 · Updated 2026-04-29

Ready for Review

Decision

Build a native SwiftUI iPhone app backed by a small Cento Mobile Gateway. Ship a local PWA link first if phone access is needed before Xcode/signing are ready.

What We Have

  • Planning packet for requirements, architecture, roadmap, validation, and notes.
  • Existing Cento surfaces for Redmine work, jobs, agents/network, dashboard activity, and notifications.
  • Mac bridge is reachable for future Xcode work.

Main Blocker

  • Linux cannot build iOS apps.
  • The Mac does not have full Xcode installed.
  • No valid signing identities are available yet.

Recommended Delivery Path

  1. Install full Xcode on the Mac and configure Apple signing/device trust.
  2. Build cento mobile serve with token auth and read-only dashboard/jobs/agents APIs.
  3. Ship the local PWA link for immediate iPhone use.
  4. Build the SwiftUI shell against fixtures, then wire it to the gateway.
  5. Validate on simulator, then physical iPhone, then use TestFlight for the cleanest install link.

First Stories

  • Mobile gateway.
  • PWA shell matching mockups.
  • SwiftUI shell.
  • Redmine activity endpoint.
  • Xcode/signing delivery.

Evidence Standard

  • Changed files.
  • Commands and tests.
  • Screenshot paths for UI.
  • Build logs for native work.
  • Known limitations.

Manager Ask

Approve PWA-first delivery and install/configure full Xcode on the Mac to unblock native iPhone install.
